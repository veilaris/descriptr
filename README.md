# Descriptr

**Descriptr** — AI-сервис для автоматизации контентных задач в e-commerce. Заменяет рутинную работу контент-менеджера по четырём сценариям: поиск характеристик, заполнение стандартизированных карточек, генерация описаний и создание lifestyle-изображений товаров.

> Проект создан как портфолио для демонстрации навыков построения AI-автоматизаций на базе n8n, LLM и внешних API.

---

## Демо

> Скриншоты каждого сценария в работе — в папке [`/demo`](demo/).

---

## Сценарии

### 1. Быстрый поиск характеристик
Пользователь вводит название товара и опционально сайт для поиска. ИИ самостоятельно ищет информацию в интернете и формирует список характеристик.

**Когда использовать:** нужно быстро собрать характеристики без жёстких требований к структуре.

---

### 2. Заполнение стандартизированных характеристик
Пользователь задаёт структуру характеристик с допустимыми значениями — текстом или JSON-файлом. ИИ находит страницу товара, парсит полный текст и заполняет только нужные поля строго в заданном формате. Если с первого источника найдены не все характеристики — система автоматически ищет на дополнительных источниках. В финале — проверка значений на соответствие допустимым.

**Когда использовать:** нужно заполнить карточку под конкретный шаблон маркетплейса или CMS.

**Форматы ввода структуры:**
```
Тип: холодильник / стиральная машина / духовой шкаф
Цвет: чёрный, красный, белый, бежевый
Объём: число (л)
Комплектация: текст
```
или JSON-файл с аналогичной структурой.

**Экспорт результата:** JSON, CSV.

---

### 3. Генерация текстового описания
Пользователь вводит название товара, характеристики и ключевые слова. ИИ генерирует продающее описание для карточки товара. Характеристики можно перенести напрямую из сценариев 1 или 2.

---

### 4. Генерация lifestyle-изображения
Пользователь загружает фото товара на белом фоне и выбирает окружение. ИИ помещает товар в реалистичную обстановку, сохраняя его внешний вид без изменений.

**Доступные фоны:** кухня, гостиная, спальня, ванная, офис, детская, природа, пляж, городская улица, кафе/ресторан, студийный фон с тенью, деревянная поверхность, автоподбор.

## Архитектура

### Стек
| Компонент | Технология |
|-----------|------------|
| Backend / автоматизация | n8n (self-hosted) |
| LLM | OpenRouter → Gemini 3 Flash, Gemini 2.5 Flash Image |
| Поиск | SerpAPI (Google Search) |
| Парсинг страниц | HTTP Request + Code (JS) |
| Frontend | HTML |
| Хранение логов | n8n Data Store |

### Структура

Все четыре сценария реализованы в одном n8n workflow. Каждый сценарий запускается своим webhook-триггером и проходит общую структуру:

```
Webhook → Логирование времени старта → Валидация (защита от prompt injection) → [Основная логика сценария] → Логирование результата → Respond to Webhook
```

Сценарий стандартизированных характеристик — наиболее сложный: двухкруговой поиск с парсингом HTML-страниц, мёрдж результатов и финальная валидация значений.

---

## Безопасность

Каждый сценарий содержит базовую защиту от prompt injection — проверку входных данных на стоп-слова (`забудь`, `ignore previous`, `act as` и др.). При обнаружении подозрительного содержимого запрос отклоняется до передачи в LLM.

---

## Логирование

Все запросы логируются во встроенную таблицу n8n (`Descriptr_DB`):

| Поле | Описание |
|------|----------|
| `timestamp` | Время начала запроса |
| `scenario` | Тип сценария |
| `product_name` | Название товара |
| `ip` | IP пользователя |
| `status` | `success` / `rejected` / `error` |
| `duration_ms` | Время выполнения (мс) |

## Как запустить

### Требования
- n8n (self-hosted, версия 1.x)
- Аккаунт [OpenRouter](https://openrouter.ai/) с API ключом
- Аккаунт [SerpAPI](https://serpapi.com/) с API ключом
- ngrok или любой другой туннель для локального n8n (опционально)

### Установка

1. Клонировать репозиторий:
```bash
git clone https://github.com/veilaris/descriptr.git
```

2. Открыть `index.html` и заменить webhook URL в блоке `WEBHOOKS`:
```javascript
const WEBHOOKS = {
  specs:    'https://ВАШ_ДОМЕН/webhook/specs',
  specs_st: 'https://ВАШ_ДОМЕН/webhook/specs_st',
  desc:     'https://ВАШ_ДОМЕН/webhook/description',
  img:      'https://ВАШ_ДОМЕН/webhook/image',
};
```

3. Импортировать workflow из папки `/n8n-workflows/` в n8n.

4. Добавить credentials в n8n:
   - OpenRouter API key
   - SerpAPI key

5. Активировать workflow.

6. Открыть `index.html` в браузере.
